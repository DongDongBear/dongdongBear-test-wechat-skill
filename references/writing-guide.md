# 写作规范

## 基本要求

- **字数**：1500-2500（短了没深度，长了跳出率高）
- **标题**：H1，20-28 个中文字符
- **段落**：每段 3-5 句，单段不超过 150 字
- **全文**：至少 3 个 H2 小标题

## 去AI痕迹（核心）

### 禁用词（出现即扣分）

首先、其次、再者、最后、综上所述、总而言之、让我们、
作为一个、不可否认、毫无疑问、事实上、众所周知、
值得注意的是、需要强调的是、在这个...的时代

### 替代策略

| 禁用 | 替代 |
|------|------|
| 首先...其次...最后 | 直接展开，用 H2 分段 |
| 作为一个XX | 我做了N年XX |
| 不可否认 | 你也发现了吧 |
| 综上所述 | 说白了就一句话 |
| 值得注意的是 | 这里有个坑 |

### 必须做到

1. **口语化**：像跟朋友聊天，不像写论文
2. **个人观点标记**："我觉得"、"我的经验是"、"说实话"
3. **不完美感**：偶尔自嘲、承认局限性
4. **具体数字**：不说"很多人"，说"身边至少 7 个朋友"
5. **画面感**：不说"效率很低"，说"每天 3 小时耗在这上面"

## 段落节奏

- 短段（1-3 句）和长段（100+ 字）交替出现
- 每 3-4 个段落设一个"钩子"（疑问句/场景切换/数据冲击）
- 不要连续超过 3 个同样长度的段落

## 开头规则（前 3 句定生死）

- 第 1 句：场景/数据/冲突（立刻抓注意力）
- 第 2 句：强化/反转（让人想继续看）
- 第 3 句：过渡到正文（给出阅读预期）

**禁止**：
- "今天给大家分享..."
- "随着XX的发展..."
- "在当今社会..."

## 结尾规则

- 用 1-2 句话总结核心观点（不要重复正文）
- 留一个思考题或行动项
- 不要用"希望对你有帮助"

## 编辑命令

用户说"润色"→ 优化表达，保持内容不变
用户说"缩写"→ 删减到原文 60-70%
用户说"扩写"→ 补充案例和细节到 130-150%
用户说"换语气"→ 调整 tone（需用户指定目标语气）
