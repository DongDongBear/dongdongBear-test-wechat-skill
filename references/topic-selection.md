# 选题评估规则

## 三维评分模型

每个选题从三个维度评分，加权计算综合分：

### 1. 热度（权重 30%）
- **9-10**: 全网爆款，微博/头条 Top 10
- **7-8**: 当日热点，至少 2 个平台上榜
- **5-6**: 行业热点，垂类用户关注
- **3-4**: 常青话题，有持续搜索量
- **1-2**: 小众话题，仅特定圈层关注

### 2. 相关度（权重 40%）
基于客户 style.yaml 的 topics 字段：
- **9-10**: 完全命中核心垂类
- **7-8**: 主垂类的延伸话题
- **5-6**: 与目标受众有交集
- **3-4**: 勉强可关联
- **1-2**: 几乎无关

### 3. 切入价值（权重 30%）
- **9-10**: 有独家视角 + 深度分析空间 + 契合 content_style
- **7-8**: 有差异化切入点
- **5-6**: 常规切入但有新信息增量
- **3-4**: 已被大量写过，难出新
- **1-2**: 纯搬运，无价值

## 输出格式

每个选题包含：
1. **标题草案**（20-28 字）
2. **综合评分**（0-10，加权计算）
3. **预计点击率**：高 / 中 / 低
4. **SEO 友好度**：引用 seo_keywords.py 的 seo_score
5. **推荐框架**：痛点型 / 故事型 / 清单型 / 对比型 / 热点解读型
6. **去重标记**：是否与 history.yaml 近期文章重复

## 排序规则

1. 先按综合评分降序
2. 同分则 SEO 友好度高的优先
3. 有"近期已覆盖"标记的自动降 2 分
