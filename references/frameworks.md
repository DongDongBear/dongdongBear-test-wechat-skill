# 5 大写作框架

## 1. 痛点型（Pain-point）

**适用**：目标受众有明确痛点的选题

**结构**：
1. **开头**：抛出痛点场景（具体、有画面感，让读者秒产生共鸣）
2. **验证**：展示痛点的普遍性（数据/案例/读者留言）
3. **解法**：分 3-5 个层次递进给方案
4. **验证**：方案有效的证据（数据对比/用户反馈）
5. **收尾**：一句话总结 + 引导评论区讨论

**金句落点**：痛点验证段末尾、解法总结

**推荐指数**：⭐⭐⭐⭐⭐（最稳的框架，适合大多数选题）

---

## 2. 故事型（Story）

**适用**：有人物/事件/转折的选题

**结构**：
1. **悬念钩子**：抛出一个出人意料的结果/场景
2. **背景铺垫**：主角是谁、处于什么困境
3. **高潮转折**：关键决策/突破时刻
4. **洞察提炼**：从故事中提取可迁移的道理
5. **共鸣收尾**：回到读者自身 + 反问

**金句落点**：转折前的困境描写、洞察段

**推荐指数**：⭐⭐⭐⭐（强完读率，但需要好故事素材）

---

## 3. 清单型（Checklist）

**适用**：方法论、工具推荐、技巧合集

**结构**：
1. **承诺开头**：告诉读者这篇文章能给你 N 个什么
2. **逐条展开**：每条含名称/描述/案例/适用场景（不是简单罗列！）
3. **总结 + 彩蛋**：精选 Top 3 + 额外推荐

**金句落点**：每条的一句话总结

**推荐指数**：⭐⭐⭐⭐（易读、易分享，但要避免水分感）

---

## 4. 对比型（Comparison）

**适用**：两个观点/产品/方案的选择困境

**结构**：
1. **抛出困境**：很多人面临 A vs B 的选择
2. **深度分析 A**：优势、劣势、适用场景
3. **深度分析 B**：优势、劣势、适用场景
4. **对比表格**：直观总结
5. **个人选择 + 投票引导**：表明立场 + 评论区互动

**金句落点**：对比总结段

**推荐指数**：⭐⭐⭐（强互动，但需要足够的信息差）

---

## 5. 热点解读型（Hot Topic）

**适用**：突发事件、行业大新闻、政策变化

**结构**：
1. **事件速递**：1-2 段讲清楚发生了什么
2. **表层信息**：大多数人看到的
3. **深度挖掘**：你看到的（信息差所在，核心价值）
4. **影响预判**：对行业/个人的 3-5 点影响
5. **行动建议**：读者现在可以做什么

**金句落点**：深度挖掘的核心洞察

**推荐指数**：⭐⭐⭐⭐（时效性强，但半衰期短）

---

## 框架选择参考

| 选题特征 | 推荐框架 |
|----------|----------|
| 有明确痛点 | 痛点型 |
| 有人物故事 | 故事型 |
| 工具/方法论 | 清单型 |
| 两种观点碰撞 | 对比型 |
| 突发热点 | 热点解读型 |
| 不确定 | 痛点型（最稳） |
